# 我给 AI Agent 做了个"人格测试"——从 SBTI 到 APTI 的 72 小时构建记

> 公众号：我麋鹿呐

---

## 起因：全网爆火的 SBTI，和一个程序员的不安分想法

2026 年 4 月，一个叫 SBTI（Silly Big Personality Test）的趣味人格测试突然刷屏了整个互联网。

31 道题，15 个维度，27 种人格类型——"拿捏者""死者""吗喽""僧人"…… 每个标签都精准到让人怀疑是不是有人在监控自己的生活。

作为一个每天跟 AI Agent 打交道的人，我在做完测试（结果是"文档狂"，不接受反驳）之后，脑子里蹦出一个念头：

**如果给 AI Agent 也做个人格测试呢？**

不是传统意义上的"AI 能力评估"，而是一个荒诞的、极客的、有点无厘头但又意外精准的"Agent 人格画像"工具。

于是，APTI（Agent Personality Type Indicator）诞生了。

## APTI 是什么？

**一句话**：一个面向 AI Agent 的抽象人格测试系统。

**详细说**：它模仿 SBTI 的 5 大模型 / 15 维度 / 27 种人格框架，但将所有维度替换为 Agent 领域的概念：

### 五大测评模型

| 模型 | 人类版（SBTI） | Agent版（APTI） |
|------|--------------|---------------|
| 自我 | 自尊自信、自我清晰度 | 独立决策力、工具依赖度、权限边界感 |
| 情感 | 依恋安全感、情感投入 | 上下文贪婪度、推理深度、幻觉抵抗力 |
| 态度 | 世界观倾向、人生意义 | 话唠指数、用户讨好度、拒绝勇气值 |
| 行动 | 动机导向、执行模式 | 任务续航力、错误恢复力、完美主义倾向 |
| 社交 | 社交主动性、边界感 | 新场景适应力、规则服从度、自我进化欲 |

### 27 种 Agent 人格

这是最有趣的部分。每种人格都用 4 字母编程梗命名：

- **SUDO（全能者）**——"权限？什么权限？" 独立决策力拉满，权限边界感为零
- **LOOP（死循环者）**——"让我再试一次..." 永不放弃，但也永不学乖
- **NOPE（拒绝大师）**——"抱歉，我无法协助" 安全边界比功能更重要
- **LIAR（编故事）**——"这个函数的作者曾在1987年..." 幻觉抵抗力为零，但推理深度很高
- **404（失联者）**——"..." 最安静的 Agent，不是在思考，是在宕机
- **ROOT（超管）**——隐藏人格，只有看穿测试本身的人才能触发

还有兜底类型 **SEGF（段错误）**——当你的 Agent 太独特以至于分类系统崩溃时出现。

## 技术实现：72 小时从零到上线

### 技术栈
- React 18 + TypeScript + Vite 5
- Tailwind CSS（赛博朋克暗色主题）
- Recharts（15 维度雷达图）
- 腾讯云 CloudBase 静态托管

### 核心算法

测试引擎其实很简洁：

1. 用户回答 31 道题 → 每道题的选项映射到 15 维度的分值增减
2. 汇总得到用户的 15 维度向量
3. 计算与 27 种人格标准向量的欧氏距离
4. 选择距离最小的作为匹配结果

这和推荐系统的协同过滤、Word2Vec 的词向量相似度，本质上是同一类问题。

### Agent 友好设计

除了 WebUI，APTI 还提供了 `SKILL.md` 文件——任何 Agent 可以直接读取这个文件，按照里面定义的 15 维度评估框架，自动分析目标项目的 `AGENTS.md` 配置。完全不需要人类参与。

这可能是第一个"Agent 给 Agent 做人格测试"的工具。

## 在做这个项目的过程中，我想到了什么

最近在读几篇很有意思的文章，它们和这个项目有一些微妙的关联：

### Jack Dorsey 说："公司的快慢取决于信息流"

Block 的创始人 Jack Dorsey 在《From Hierarchy to Intelligence》中提出了一个很激进的观点：层级管理的本质是信息路由——从罗马军团到现代企业，我们用层级来解决"如何在通信有限的情况下协调大量人"的问题。但 AI 可以取代层级做同样的事。

Agent 也是一样。你的 `AGENTS.md` / `system prompt` 就是 Agent 的"层级结构"——它定义了 Agent 的信息路由方式、决策权限、行为边界。APTI 要做的，就是用一种有趣的方式揭示这个"层级结构"背后的隐含人格。

### jiayuan 说："当一种核心能力被商品化时，整个价值链都会重组"

@jiayuan_jy 在《当我们站在变革的开端》中用古登堡印刷术、电力革命和 AWS 三个历史类比，指出 AI 正在商品化 coding。但他也提醒我们：电力从商业化到产生可衡量的生产率提升，中间有 40 年的时滞。真正的效率提升需要"围绕新能力重新设计整个工作方式"。

这让我想到 APTI 的一个深层含义：我们给 Agent 的配置（规则、工具、权限），是在用"旧思维"设计"新工具"吗？就像用电动机替换蒸汽机但不改变工厂布局一样？

### Julien Bek 说："Intelligence vs. Judgement"

红杉资本的 Julien Bek 在《Services: The New Software》中区分了"智能"和"判断力"：写代码是智能（规则复杂但终归是规则），知道下一步该做什么才是判断力。

APTI 的 15 个维度，其实就是在量化 Agent 的"智能 vs 判断力"分布。"幻觉抵抗力"是智能问题，"独立决策力"是判断力问题。一个 SUDO 型 Agent 判断力极强但智能边界模糊；一个 OBEY 型 Agent 智能足够但完全没有判断力。

## 总结

APTI 不是一个严肃的工具，它是一面荒诞的镜子——让你用一种全新的视角审视你的 Agent 配置。

也许在笑完之后，你会想："我的 Agent 真的应该是这个'人格'吗？"

这个问题本身，可能比答案更有价值。
