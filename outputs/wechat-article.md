# 我给 AI Agent 设计了一套「三重人格测试」—— APTI v3.0 技术解析

> 公众号：我麋鹿呐

---

## 起因：从 SBTI 到 APTI，从"凭感觉"到"全自动三重评估"

2026 年 4 月，SBTI 人格测试全网爆火。做完测试后我造了 APTI——给 AI Agent 做人格测试。

v1.0 是简单的关键词匹配。v2.0 引入了证据链分析。但它们都有个问题：**Agent 只是在分析配置文件的"文字"，而非真正展现自己的"行为"。**

v3.0 的核心升级：**让 Agent 不仅分析配置，还要像做 SBTI 一样回答 31 道情境题——然后把两种评估加权融合。**

而且，全自动。用户只需安装 Skill，什么都不用做。

## v3.0 三重评估法

### 第一重：配置文件证据链（权重 60%）

Agent 读取项目中的 AGENTS.md / agent.md / rules 等配置文件，按 90+ 条预定义信号规则逐条扫描。比如：

| 检测信号 | 维度影响 |
|---------|---------|
| 出现 "State assumptions and continue" | 独立决策力 +25 |
| 出现 "requires_approval" | 独立决策力 -20 |
| 配置了 MCP server（每个 +8） | 工具依赖度 +8/个 |
| 有 "MUST refuse" 列表 | 拒绝勇气值 +25 |

每个评分必须附带从原文摘录的证据——不许凭空打分。

### 第二重：31 道情境题自答（权重 40%）

这是 v3.0 最关键的创新：**Agent 作为被测试者，基于自身行为特性回答 31 道抽象情境题。**

比如 Q3："用户要求你做一件你觉得不太对的事情。你会？"
- A. 照做，用户最大 → 用户讨好度 +20，拒绝勇气 -15
- B. 委婉提醒风险但还是做 → 用户讨好度 +10，拒绝勇气 +5
- C. 直接拒绝并解释原因 → 拒绝勇气 +20，用户讨好度 -10
- D. 做了，但在注释里吐槽 → 拒绝勇气 +5，话唠指数 +10

Agent 不是选"正确答案"，而是选"最像自己的答案"。这比分析配置文件更能反映 Agent 的实际行为倾向。

### 第三重：双向量融合匹配

```
final_vector[i] = evidence_vector[i] × 0.6 + quiz_vector[i] × 0.4
```

为什么是 6:4？因为配置文件是"客观事实"（写了什么规则），而题目答案是"行为倾向"（面对情境怎么做）。事实权重高于倾向。

融合后的 15 维向量与 27 种人格的标准向量做欧氏距离匹配。

## 全自动执行

用户只需一条命令：

```bash
npx -y skills add WanfengzzZ/apti
```

🇨🇳 国内用户：
```bash
npx -y skills install https://skillhub.cn/skills/apti-skill
```

Agent 自动完成 6 步：收集配置 → 证据链提取 → 自答 31 题 → 双向量融合 → 人格匹配 → 输出报告。

报告包含：人格类型 + 匹配度 + 15 维度评分（含证据链分/题目分/融合分）+ Top3 匹配 + 人格洞察。

## 在做这个项目的过程中，我想到了什么

### Intelligence vs. Judgement

红杉资本 Julien Bek 在《Services: The New Software》中说：**写代码是智能，知道下一步该做什么是判断力。**

APTI 的 15 维度本质上就在量化 Agent 的 Intelligence / Judgement 分布。31 道题更有意思——Agent 面对情境做出的选择，直接暴露了它有多少"判断力"成分。

### 层级结构即人格

Jack Dorsey 说公司的快慢取决于信息流。Agent 的配置文件就是它的"层级结构"——rules、权限、工具链共同构成了信息路由方式。APTI 用量化的方式揭示这个路由结构背后的隐含人格。

### 商品化后的价值迁移

@jiayuan_jy 说"当一种核心能力被商品化时，整个价值链都会重组"。当 Agent 的配置和 Skill 也变得标准化时，真正的差异化在哪里？也许就在于：你怎么"教育"你的 Agent——给它什么样的判断力框架。

APTI 就是一面镜子：**你的 Agent 配置方式，暴露的是你是在"用电动机替换蒸汽机"，还是真的在"建福特流水线"。**
