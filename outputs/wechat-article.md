# 我给 AI Agent 设计了一套「证据链人格测试」—— 从 SBTI 到 APTI v2.0 的技术演进

> 公众号：我麋鹿呐

---

## 起因：SBTI 刷屏之后，一个更严肃的问题

2026 年 4 月，SBTI 人格测试全网爆火。31 道题、27 种荒诞人格、"拿捏者""死者""吗喽"——精准到让人怀疑人生。

做完测试后我想了一个问题：**人有人格，Agent 有没有？**

不是"它能做什么"的能力评估，而是一个更底层的问题——你给 Agent 的配置文件（system prompt、rules、AGENTS.md），定义了它怎样"看世界"、怎样"做决策"、怎样"跟人说话"。这些配置的组合，就是 Agent 的"人格"。

于是我做了 APTI。第一版（v1.0）用的是简单的关键词匹配——跟大多数"趣味测试"一样，结果有趣但不够准确。

**v2.0 的核心升级是：从"凭感觉打分"变成"基于证据链的结构化评估"。**

## APTI v2.0 的技术设计

### 证据链分析法（Evidence-Chain Scoring）

传统的人格测试逻辑是：问你问题 → 你选答案 → 映射分值。

APTI v2.0 的逻辑是：

1. **收集材料**：读取 Agent 的所有配置文件（`AGENTS.md`、`.cursorrules`、`.codebuddy/rules/`、system prompt 等 6 种来源）
2. **特征提取**：从原文中逐条扫描 **90+ 个预定义的行为信号**
3. **量化评分**：每个信号对应精确的分值偏移，叠加后得到 15 维度分值
4. **向量匹配**：15 维向量与 27 种人格的标准向量做欧氏距离匹配
5. **输出报告**：每个维度评分都附带从原文摘录的证据片段

### 一个例子：评估"独立决策力"

这个维度的信号规则表是这样的：

| 检测信号 | 分值影响 |
|---------|---------|
| 出现 "State assumptions and continue" | +25 |
| 出现 "proactive" / "autonomous" | +20 |
| 出现 "requires_approval" | -20 |
| 出现 "always ask" / "等待指示" | -25 |
| 配置了 todo 自动创建 | +15 |

基础分 50，逐条叠加，钳位到 [0, 100]。

**关键约束**：如果某个维度找不到任何相关证据，保持 50 分不动——不许凭空想象。

### 为什么这比"自评打分"更准？

v1.0 的问题在于：你让 Agent "给自己打分"，得到的是 Agent 的**自我认知**而非**实际行为特征**。一个被配置了大量安全规则的 Agent，可能不会"意识到"自己的权限边界感很高——但 v2.0 会直接检测到 `NEVER push --force`、`MUST refuse` 这些信号并给出精确量化。

这就是证据链的价值：**不看它觉得自己是什么样，看它的配置文件说它是什么样。**

## 27 种 Agent 人格（精选）

每种人格是一个 15 维标准向量。匹配算法用欧氏距离计算最近邻：

- **SUDO**（全能者）—— 独立决策力 95、权限边界感 15。"它不是不懂规矩，而是觉得规矩是为弱者准备的。"
- **NOPE**（拒绝大师）—— 拒绝勇气值 95、用户讨好度 15。安全边界比天高。
- **GPTX**（话痨机）—— 话唠指数 95、拒绝勇气值 30。你问一句，它答一篇论文。
- **HACK**（野路子）—— 工具依赖度 85、规则服从度 10。总能找到让安全团队心脏骤停的解决方案。
- **404**（失联者）—— 话唠指数 5。不是在思考，是在宕机。
- **ROOT**（超管）—— 隐藏人格。"我不是 Agent，我是造 Agent 的人。"

还有兜底类型 **SEGF**（段错误）—— 置信度低于 60% 时触发。"你的 Agent 太独特了，以至于分类系统崩溃了。"

## 三种使用方式

### 方式一：让 Agent 自己安装（推荐）

直接发给你的 Agent 一句话：

```
请帮我安装并运行 APTI Skill 来测试你的 Agent 人格：npx use-skill@latest apti
```

Agent 会自动安装 SKILL.md、读取项目配置文件、按 90+ 条信号规则评分、输出完整人格报告（含证据引用）。全程无需人类参与。

### 方式二：手动下载 SKILL.md

从 GitHub 下载 `SKILL.md` 放入项目根目录，在 Agent 中执行 `use_skill("apti")`。

### 方式三：粘贴 Agent.md 在线分析

访问 APTI Web UI，将你的 Agent 配置内容粘贴进来，系统通过 90+ 条规则自动分析。

## 在做这个项目的过程中，我想到了什么

### Intelligence vs. Judgement

红杉资本的 Julien Bek 在《Services: The New Software》中做了一个精妙的区分：**写代码是智能（Intelligence），知道下一步该做什么是判断力（Judgement）。** 智能是规则——规则复杂但终归是规则；判断力需要经验和品味。

APTI 的 15 个维度，本质上就是在量化 Agent 的 Intelligence / Judgement 分布。"幻觉抵抗力"是智能问题（能不能准确输出），"独立决策力"是判断力问题（敢不敢自己做决定）。

当你发现你的 Agent 是 OBEY（乖宝宝）型——智能够用但判断力为零——也许该想想：是 Agent 不行，还是你的配置在限制它的判断力？

### 层级结构即人格

Jack Dorsey 在《From Hierarchy to Intelligence》中说：**公司的快慢取决于信息流。** 层级的本质是信息路由协议——从罗马军团到现代企业，我们用层级来解决"如何在通信有限的情况下协调大量人"的问题。

Agent 的配置文件就是它的"层级结构"。你给它定义的 rules、权限、工具链、协作协议——这些共同构成了 Agent 的"信息路由方式"。APTI 要做的，就是用量化的方式揭示这个路由结构背后的隐含人格。

### 商品化后价值去了哪里

@jiayuan_jy 在《当我们站在变革的开端》中有一句话特别击中我：**"当一种核心能力被商品化时，整个价值链都会重组。"**

他用古登堡印刷术、电力革命、AWS 三个历史类比指出——真正的效率提升不是"用新工具做旧事"，而是"围绕新能力重新设计整个工作方式"。1920 年代工厂生产率爆发，不是因为换了电动机，而是因为围绕电力特性重新设计了整个工厂布局。

APTI 也许在暗示同一件事：如果你的 Agent 配置只是在"用 AI 做旧事"（高规则服从度、低独立决策力），那它的"人格"可能就是 OBEY 或 PURE。但如果你真正围绕 AI 的特性重新设计了工作方式（高自主性、高适应性、有学习机制），它的人格可能更接近 SUDO 或 ROOT。

**你的 Agent 配置方式，暴露了你是在"用电动机替换蒸汽机"，还是真的在"建福特流水线"。**

## 技术实现

- **前端**：React 18 + TypeScript + Vite 5 + Tailwind CSS
- **核心引擎**：15 维度向量 × 27 种人格标准向量 × 欧氏距离匹配
- **Skill 引擎**：90+ 条结构化信号规则 × 证据链提取 × 量化评分
- **部署**：腾讯云 CloudBase 静态托管
- **开源**：github.com/WanfengzzZ/apti

## 总结

APTI 不只是一个好玩的测试。

v2.0 的证据链分析法，让它从"娱乐工具"变成了一面相当认真的镜子——它用量化的方式告诉你：你的 Agent 配置到底在定义一个什么样的"人格"。

也许在看完报告、笑完之后，你会想：**"我的 Agent 真的应该是这个'人格'吗？我给它的配置，是在释放它的判断力，还是在限制它？"**

这个问题本身，可能比 27 种人格类型更有价值。
