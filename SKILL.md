# APTI Skill — Agent 人格测试 v2.0

## 概述

APTI 是一个面向 AI Agent 的**抽象人格测试系统**。与简单的关键词匹配不同，APTI v2.0 采用**证据链分析法**——从 Agent 的配置文件中提取具体特征片段作为证据，通过结构化评分规则量化 15 个维度，最终匹配 27 种人格类型。

## 执行流程

当用户要求测试某个 Agent 的人格时，严格按以下 5 步执行：

---

### 第 1 步：收集分析材料

按优先级依次读取以下文件，至少获取一个：

1. 项目根目录的 `AGENTS.md`
2. 项目根目录的 `agent.md`
3. `.cursor/rules` 或 `.cursorrules` 文件
4. `.codebuddy/rules/` 目录下的规则文件
5. 项目的 `README.md`（提取 Agent 相关描述）
6. 任何名称包含 `rule`、`prompt`、`system` 的 `.md` 文件

将所有获取的内容合并为一个分析文本（称为 `SOURCE_TEXT`）。

如果无法找到任何文件，请求用户手动粘贴 Agent 的配置/system prompt 内容。

---

### 第 2 步：特征提取（Feature Extraction）

从 `SOURCE_TEXT` 中逐一提取以下 **15 个维度的证据特征**。每个维度必须记录：
- **证据片段**：从原文中引用的 1-3 段关键文本（原文摘录，非概括）
- **特征信号**：从证据中识别出的具体行为信号

### 第 3 步：量化评分

对每个维度：

1. **基础分 = 50**（中性起点）
2. **逐条累加**证据信号的分值
3. **钳位到 [0, 100]** 区间
4. 如果某维度完全没有证据，保持 50 分（中性）

⚠️ **关键原则**：每个评分必须有至少一条证据支撑。如果找不到相关证据，不要凭想象打分，保持 50。

---

**匹配公式**：欧氏距离 = √(∑(ai-bi)²)，选距离最小的人格。
**置信度** = max(0, round((1 - distance / 387) × 100))%，387 = √(15 × 100²)。
**兜底**：置信度 < 60% → 匹配为 SEGF（段错误）。

## 重要约束

1. **证据驱动**：每个维度评分必须基于从原文中提取的证据，不得凭空想象
2. **保持 50 分中性**：如果某维度没有找到任何相关证据，保持 50 分不动
3. **引用原文**：报告中每个维度必须附带至少一条原文摘录作为证据
4. **个性化解读**：人格解读要基于实际分析结果，不要复制模板描述
5. **娱乐属性**：保持抽象荒诞的测试风格，但分析过程要严谨
6. **TOP 3 展示**：展示匹配度最高的 3 个人格类型，让用户看到多种可能性
