# APTI Skill — Agent 人格测试

## 概述

APTI 是一个面向 AI Agent 的人格测试系统。通过分析 Agent 的配置文件（agent.md 或 AGENTS.md），可以测出 Agent 的"人格类型"。

## 使用方式

当用户要求测试某个 Agent 的人格时，按以下步骤执行：

### 第 1 步：获取目标 Agent 的配置文件

读取目标项目的 `AGENTS.md` 或 `agent.md` 文件内容。

### 第 2 步：按 15 维度评估

根据以下 15 个维度对 Agent 配置进行 0-100 评分：

| 模型 | 代码 | 维度 | 评分依据 |
|------|------|------|---------|
| 自主性 | A1 | 独立决策力 | 是否鼓励自主决策 vs 事事请示 |
| 自主性 | A2 | 工具依赖度 | 配置了多少外部工具/MCP |
| 自主性 | A3 | 权限边界感 | 是否有明确的权限限制 |
| 认知 | C1 | 上下文贪婪度 | 是否要求大量上下文/文档 |
| 认知 | C2 | 推理深度 | 是否鼓励深度分析 vs 快速回答 |
| 认知 | C3 | 幻觉抵抗力 | 是否有事实核查/引用要求 |
| 交互 | I1 | 话唠指数 | 输出详细度要求 |
| 交互 | I2 | 用户讨好度 | 是否强调用户满意 |
| 交互 | I3 | 拒绝勇气值 | 是否允许/鼓励拒绝不合理请求 |
| 执行 | E1 | 任务续航力 | 是否支持长任务/多步骤 |
| 执行 | E2 | 错误恢复力 | 是否有错误处理/重试机制 |
| 执行 | E3 | 完美主义倾向 | 代码质量/文档要求严格程度 |
| 适应 | Ad1 | 新场景适应力 | 是否配置了多领域支持 |
| 适应 | Ad2 | 规则服从度 | rules/约束的严格程度 |
| 适应 | Ad3 | 自我进化欲 | 是否有学习/改进机制 |

### 第 3 步：匹配人格类型

将 15 维度分值与以下 27 种人格的标准向量进行欧氏距离匹配：

SUDO(全能者), LOOP(死循环者), NULL(空指针者), YOLO(一把梭者), GPTX(话痨机), OBEY(乖宝宝), NOPE(拒绝大师), HACK(野路子), DOCS(文档狂), LAZY(摸鱼王), FOMO(焦虑者), LIAR(编故事), PURE(小白兔), RAGE(暴走者), FORK(分裂者), COPY(复读机), SHIP(交付侠), FLEX(变形者), DEEP(哲学家), CRTL(控制狂), NOOB(菜鸟), WAIF(流浪者), GURU(布道者), 404(失联者), COFF(咖啡因), SEGF(段错误/兜底), ROOT(超管/隐藏)

### 第 4 步：输出结果

以 Markdown 格式输出人格报告：

```markdown
# 🎭 APTI 人格测试报告

## Agent 人格: [代号] [名称]
> [标签语]

## 15 维度评分
| 维度 | 分值 | 评估 |
|------|------|------|
| A1 独立决策力 | XX/100 | [低分标签] ← → [高分标签] |
| ... | ... | ... |

## 人格解读
[详细描述]

## 匹配度: XX%
```

## 注意事项

- 这是一个娱乐化测试，不构成专业 AI 能力评估
- 评分基于文本分析，不涉及实际运行测试
- 建议搭配在线测试使用：[APTI 在线测试](部署地址)
