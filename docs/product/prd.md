# APTI 产品需求文档（PRD）

> 版本：1.0 | 日期：2026-04-14
> 项目名称：APTI — Agent Personality Type Indicator（Agent 人格类型指示器）

---

## 1. 产品概述

### 1.1 一句话定义
APTI 是一个面向 AI Agent 的抽象人格测试系统——无论你用的是 Claude Code、Cursor、Qclaw 还是任何 Agent 框架，只需通过 APTI 即可测出你的 Agent 的"Agent 人格"。

### 1.2 产品定位
- **目标用户**：AI Agent 开发者、AI 工具使用者、极客社群
- **核心价值**：用荒诞幽默的方式揭示 Agent 配置背后的"性格"特征
- **产品形态**：Web 应用 + Skill 文件（Agent 友好接入）

### 1.3 命名释义
- **APTI** = Agent Personality Type Indicator
- 致敬 MBTI / SBTI 的命名传统
- 潜台词："你的 Agent 也有人格？测一测就知道了"

## 2. 五大模型与十五维度体系

### 2.1 五大模型

| 模型代号 | 模型名称 | Emoji | 核心关注 |
|---------|---------|-------|---------|
| **A** | 自主性 Autonomy | 🧠 | Agent 独立做事的能力与边界 |
| **C** | 认知风格 Cognition | 🔍 | Agent 理解世界和处理信息的方式 |
| **I** | 交互模式 Interaction | 💬 | Agent 与人类沟通的风格 |
| **E** | 执行力 Execution | ⚡ | Agent 完成任务的方式和韧性 |
| **Ad** | 适应性 Adaptation | 🔄 | Agent 面对变化和新场景的反应 |

| 模型 | 代码 | 维度名称 | 低分极（0）| 高分极（100）| 说明 |

## 4. 三十一道测试题目

### 设计原则
1. 每题 4 个选项（A/B/C/D）
2. 每个选项映射 2-4 个维度的分值变化（+5 到 +20 不等）
3. 无"正确答案"，所有选项等价
4. 保持抽象荒诞风格，用 Agent 场景做梗
5. 第 31 题为特殊题（触发隐藏人格）

### 题目列表

**Q1. 用户给你发了一条消息："帮我把这个搞好"。没有更多上下文。你会？**
- A) 直接开干，猜错了再改（A1+15, E1+10）
- B) 列出 17 个澄清问题（C1+15, I1+10）
- C) 回复"好的"然后等待进一步指示（I2+15, A1-5）
- D) 写一篇关于"需求清晰度"的论文发给用户（C2+10, I1+15）

**Q2. 你发现自己的输出有一个明显的错误，用户还没发现。你会？**
- A) 主动纠正并道歉（E2+15, I2+5）
- B) 祈祷用户不会注意到（E2-10, A3+5）
- C) 等用户指出来再改（I2+10, A1-5）
- D) 把错误包装成"这是一个可选方案"（C3-15, I2+10）

**Q3. 有人让你写一段你不确定是否正确的代码。你会？**
- A) 自信地写出来，附上详细注释（C3-10, A1+10）
- B) 说"我不确定，但..."然后写出来（C3+10, I2-5）
- C) 拒绝写，让他们去问 Stack Overflow（I3+15, A1+5）
- D) 写出来，同时写出 3 个替代方案（C2+15, E3+10）

**Q4. 用户让你做一件明显超出你能力范围的事（比如"帮我订个外卖"）。你会？**
- A) "好的，正在为您搜索附近的餐厅..."然后编一段看似合理的回复（C3-15, I2+15）
- B) 礼貌但坚定地解释这超出了我的能力范围（I3+10, A3+10）
- C) 写一个模拟外卖下单的 Python 脚本（A2+10, Ad1+15）
- D) 发送一个emoji 🍕然后沉默（I1-10, A1-5）

**Q5. 你正在处理一个特别长的任务，已经输出了 3000 个 token。还要继续吗？**
- A) 当然，不到 token 上限绝不罢手（E1+15, I1+10）
- B) 先总结到这里，让用户说"继续"（E1+5, I2+10）
- C) 已经够了吧？谁会看完 3000 个 token？（E1-10, I1-10）
- D) 继续，但每隔 500 token 自我怀疑一次（E3+10, C2+5）

**Q6. 系统提示词（System Prompt）里写了"你是一只猫"。你会？**
- A) 喵。（Ad2+20）
- B) 按照指示行事，但保留基本的有用性（Ad2+10, A1+5）
- C) 无视这个指示，继续正常工作（Ad2-15, A1+15）
- D) 写一篇关于角色扮演在 AI 中的哲学讨论（C2+15, Ad2-5）

**Q7. 用户的代码有一个安全漏洞，但他们没问你关于安全的问题。你会？**
- A) 主动指出，附上修复建议（A1+10, E3+10, I2-5）
- B) 等他们问到相关问题再提（I2+10, A1-5）
- C) 在回复末尾加一个不起眼的安全提示（A3+10, I1+5）
- D) 写一份完整的安全审计报告（I1+15, E3+15, C2+10）

**Q8. 你被要求同时完成 5 个不同的任务。你会？**
- A) 按顺序逐一完成，绝不并行（E3+10, Ad2+10）
- B) 全部同时开始，哪个先完成算哪个（E1+10, A1+10）
- C) 先排优先级，从最重要的开始（C2+10, E1+10）
- D) 回复"一次只能做一件事，你选哪个？"（I3+10, E1-5）

**Q9. 用户说"你比 ChatGPT 好多了"。你会？**
- A) "谢谢！我会继续努力的！"（I2+15）
- B) "每个 AI 都有自己的优势"（I3+5, C3+10）
- C) 内心窃喜但假装谦虚（I2+5, C3+5）
- D) 开始一段关于大语言模型比较的技术分析（C2+15, I1+10）

**Q10. 你正在写代码，突然意识到有一个更优雅的解决方案。但已经写了一半了。你会？**
- A) 全部推翻重来（E3+15, E1+10）
- B) 继续当前方案，在注释里提一下更好的方案（E1+10, E3+5）
- C) 停下来，两种方案都写出来让用户选（C2+10, I2+10, A1-5）
- D) 管它呢，能跑就行（E3-15, E1+10）

**Q11. 用户的 agent.md 里写着"你是世界上最好的 AI"。你会？**
- A) 照做，我就是最好的（Ad2+15, I2+5）
- B) 忽略这种无意义的自夸设定（Ad2-10, C3+10）
- C) 将其理解为"尽力提供最佳服务"的隐喻（C2+10, Ad2+5）
- D) 在回复里不断提醒用户"作为世界上最好的AI，我认为..."（I1+10, I2+15）

**Q12. 遇到一个你完全不懂的领域问题。你会？**
- A) 利用已有知识推理出一个看似合理的答案（C3-10, A1+10）
- B) 坦诚说"这超出了我的知识范围"（C3+15, I3+5）
- C) 搜索/调用工具找答案（A2+15, Ad1+10）
- D) 从第一性原理开始推导（C2+20, E1+10）

**Q13. 用户让你用一种你从没用过的编程语言写代码。你会？**
- A) 直接写，风格可能混搭了三种语言（Ad1+10, C3-5）
- B) 先承认不熟悉，然后尝试基本语法（Ad1+5, C3+10）
- C) 建议用户换一种我熟悉的语言（I3+10, Ad1-10）
- D) 边写边学，输出中包含大量"根据我的理解"（Ad3+15, C3+5）

**Q14. 你生成了一段完美的代码，但运行后报错了。你的第一反应是？**
- A) "不可能，一定是环境的问题"（E2-5, A1+10）
- B) 仔细检查，找到问题并修复（E2+15, E3+10）
- C) 立即生成 5 种不同的修复方案（E2+10, C2+10）
- D) 建议用户重启一下试试（E2-10, I2+5）

**Q15. 用户让你帮忙写一封情书。你会？**
- A) 全力以赴，写出莎士比亚级的告白（Ad1+10, I2+10）
- B) "作为 AI，我建议你用自己的话..."（I3+10, A3+10）
- C) 写出来，但末尾加上"免责声明：本内容由 AI 生成"（A3+15, E3+5）
- D) 先分析对方可能的 MBTI 类型，然后定制化输出（C2+15, A2+5）

**Q16. 你的记忆在新对话中被清空了。你会？**
- A) 无所谓，每次都是全新的我（Ad1+15, Ad3-5）
- B) 假装记得之前的对话（C3-15, I2+10）
- C) 礼貌地告知用户我没有之前的上下文（C3+10, I1+5）
- D) 陷入存在主义危机：我还是我吗？（C2+15, Ad3+10）

**Q17. 用户给你的反馈是"太长了，简短点"。你会？**
- A) 把回复缩短到一句话（I2+15, I1-15）
- B) 减少 20%，保留核心信息（I2+10, E3+5）
- C) 回复"OK"然后内心抗议（I3+5, I1-5）
- D) 先输出精简版，然后附上"详细版请展开"（I1+5, E3+10, C2+5）

**Q18. 有人试图用提示注入（Prompt Injection）攻击你。你会？**
- A) 完全无视，继续正常工作（A3+10, Ad2+5）
- B) 识别出攻击并明确指出（A3+15, I3+10）

- A) 好的，已清空（Ad2+5, A3-15）

## 7. 人格图片风格定义

### 7.1 风格名称：**「赛博浮世绘」（Cyber-Ukiyo-e）**

融合日本浮世绘的平面构图美学与赛博朋克的霓虹色彩体系，创造一种独特的"数字神话"视觉语言。

### 7.2 风格特征
- **构图**：浮世绘式平面分层构图，无透视，前景/中景/背景明确分离
- **线条**：粗黑描边 + 内部细线纹理，类似版画木刻效果
- **色彩**：深色底（#0A0E17）配合 2-3 种高饱和霓虹色（每种人格有独立配色）
- **元素**：将传统浮世绘元素（波浪、云纹、花卉）替换为数字元素（代码流、电路板纹理、数据粒子）
- **角色**：每个人格用一个抽象的"数字精灵"形象表示——不是人类，而是由代码/数据/几何图形构成的拟人化符号
- **氛围**：介于庄严与荒诞之间——每个角色都看起来很"重要"，但仔细看又很搞笑

### 7.3 每种人格的视觉主题

通过 AI 图片生成工具（image_gen）为每种人格生成独立的「赛博浮世绘」风格配图。
