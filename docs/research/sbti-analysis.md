## 2. 五大模型与十五维度

### 2.1 模型结构

SBTI 采用 5 大心理模型，每个模型下设 3 个评估维度：

| 模型 | Emoji | 维度代码 | 维度名称 |
|------|-------|---------|---------|
| 🪞 **自我模型** | Self | S1 | 自尊自信 |
| | | S2 | 自我清晰度 |
| | | S3 | 核心价值 |
| 💔 **情感模型** | Emotion | E1 | 依恋安全感 |
| | | E2 | 情感投入度 |
| | | E3 | 边界与依赖 |
| 🎭 **态度模型** | Attitude | A1 | 世界观倾向 |
| | | A2 | 规则与灵活度 |
| | | A3 | 人生意义感 |
| ⚡ **行动模型** | Action | Ac1 | 动机导向 |
| | | Ac2 | 决策风格 |
| | | Ac3 | 执行模式 |
| 🎪 **社交模型** | Social | So1 | 社交主动性 |
| | | So2 | 人际边界感 |
| | | So3 | 表达与真实度 |

### 2.2 评分机制

- 每道题选项映射到多个维度的分值增减
- 15 维度得分归一化后形成用户向量
- 通过与 27 种人格标准向量的距离计算进行匹配
- 匹配度低于 60% 触发兜底类型
- 特定选项组合触发隐藏类型

## 4. 题目设计特点

### 4.1 题目风格
- **情境模拟**：将被测者置于具体场景中做选择
- **反逻辑提问**：故意设计违反常识的选项，测试真实反应
- **元提问**：关于测试本身的问题（"你觉得这个测试准吗？"）
- **荒诞选项**：每道题至少有一个荒诞/搞笑选项

### 4.2 设计原则
1. 避免"政治正确"选项——没有明显的"好答案"
2. 选项之间差异足够大，避免模糊地带
3. 每道题至少关联 2-3 个维度的分值变化
4. 题目之间有交叉验证机制

## 5. 技术实现分析

### 5.1 架构
- 单页 HTML 静态站点
- 所有逻辑在前端 JavaScript 完成
- 无后端服务器，无网络请求
- 图片资源本地存储

### 5.2 核心算法
1. 用户回答 31 题 → 每题选项产生 15 维度分值变化
2. 汇总 15 维度得分并归一化到 [0, 100]
3. 计算用户向量与 27 种人格标准向量的欧氏距离
4. 选择距离最小的类型作为结果
5. 匹配度 < 60% → 触发兜底类型 HHHH
6. 特定题目组合 → 触发隐藏类型 DRUNK

### 5.3 结果展示
- 15 维度雷达图（可视化）
- 人格类型卡片（配图 + 解读）
- 一键分享卡片功能（html2canvas）

## 6. 传播学分析

### 6.1 爆火因素
1. **标签化社交货币**：27 种荒诞人格标签天然具有传播性
2. **低门槛高趣味**：3 分钟完成，结果引发共鸣或争议
3. **反 MBTI 定位**：对严肃人格测试的解构和戏谑
4. **分享机制**：一键生成分享卡片，适配社交媒体

### 6.2 用户动机
- **自我确认**：通过标签获得身份认同
- **社交破冰**：互测互比促进关系
- **娱乐解压**：荒诞风格提供情绪价值
